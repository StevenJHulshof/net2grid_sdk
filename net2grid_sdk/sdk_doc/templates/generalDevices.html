generalDevices

The generalDevices object contains a list of all general devices found in the ZigBee network. Each device has the following methods available.

Methods

Name Type Details
logAvailableAttributes() Function The developer can call this method to print a list in the browser console of the common available attributes of the clusters present in the virtual device. To specify the attribute set of the device, use the discoverAttributes(clusterId) method. The logAvailableAttributes() method can be used as a development tool.
logAvailableCommands() Function This method does the same as the logAvailableAttributes() method, but prints the common available commands of the clusters present in the virtual device. This method can also be used as a development tool.
logDeviceInfo() Function The logDeviceInfo() method will print information about the virtual device in the browser console. This method can also be used as a development tool.
checkCluster(clusterId) Function This method will check whether the input cluster parameter is available in the virtual device. Useful for checking the functionality of the device.
writeCommand(command, args) Function The writeCommand(command, args) method can be used to send cluster specific commands to the actual devices.
readAttributes(clusterId, attributes) Function This method can be used to read the actual value of one or multiple attributes within a device.
writeAttributes(clusterId, attributes, value) Function If an attribute is writable, the writeAttributes(clusterId, attributes, value) method can be used to directly write values in these attributes.
discoverAttributes(clusterId) Function For a more specific attribute list of a device, the discoverAttributes(clusterId) method can be called. This method can only return an attribute list of one cluster at a time.
configureReportingSend(clusterId, attributes, timeouts) Function This method can be used if the developer is intending to send reports to the target device.
configureReportingReceive(clusterId, attributes, minInterval, maxInterval, reportableChange) Function This method can be used if the developer wishes to receive reports from the target device.
readReportingConfiguration(clusterId, attributes, directions) Function The readReportingConfiguration(clusterId, attributes, direction) method can be used if the developer wants to know the details of the reporting configuration of an attribute.
